# 国公費マスタ（地単公費）：システム開発観点での構造整理

> **整理の軸：** 何を入力として受け取り、何が出力として決まるのか。  
> 入力は「可否判定に使う引数」「計算に使う引数」「処理に無関係なメタ情報」の3種に分類。

---

## 📤 出力（計算結果として決まるもの）

| 出力 | 内容 |
|------|------|
| **適用可否** | この制度が当該患者・レセプトに適用されるか（true/false） |
| **公費負担額 / 患者自己負担額** | 実際に公費が負担する金額、または患者が窓口で払う金額（円） |
| **助成方法** | 現物給付（レセプト併用 or 連記式）または償還払い |
| **食事・生活療養費の患者負担額** | 食事療養・生活療養に関する患者負担分 |

---

## 🔍 入力①：適用可否の判定（Eligibility Inputs）

> 「この制度は誰に・いつ・どの保険で適用されるか？」を決めるフラグ群

| 項番 | 項目名 | 説明 |
|------|--------|------|
| 10 | 有効開始年月日 | 制度が適用開始となる年月日 |
| 11 | 有効終了年月日 | 制度が適用終了となる年月日（未定の場合 99991231） |

| 14 | 年齢基準（未満） | 年齢上限の基準日の種類（年度末/月末/前日 など7種） |
| 15 | 対象年齢（未満） | 上記基準日で○歳未満なら対象 |

| 18 | 社保 | 社保加入者が対象か |

| 20 | 後期高齢 | 後期高齢者医療加入者が対象か |

### 対象者の属性・状態

## 📋 メタ情報（計算に非使用）

> システムの処理ロジックには影響せず、管理・表示・検索などに用いる情報

| 項番 | 項目名 | 説明 |
|------|--------|------|
| 1 | 事業名（正式名称） | 地単公費の正式名称（表示・印刷用） |
| 2 | 事業名（略称） | 略称（表示・UI用） |
| 3 | 都道府県番号 | 登録主体の識別。ただし市区町村コード未入力時に都道府県単独事業と判断する副作用あり ⚠️ |
| 4 | 市区町村コード | 登録主体の識別。未入力=都道府県単独事業として計算優先される副作用あり ⚠️ |
| 5〜7 | 分類番号（1〜3） | 政策分類（こども/難病/障害者 等）。全国共通の優先順位自動判定に使用 ⚠️ |
| 8 | 公費負担者番号 | 受給者証の8桁番号（識別子） |
| 9 | 事業内区分コード | 同一公費負担者番号内の行番号（連番） |
| 22_2 | 所得区分名称（独自定義分） | A0〜ZZ選択時の所得区分ラベル（PMH連携用テキスト） |
| 79 | その他制度に係る参考情報 | **明示的に「計算に利用しない」** と仕様書に記載された備考欄 |

> ⚠️ **注意：** 都道府県番号・市区町村コード・分類番号は「メタ情報」的な見た目だが、計算優先順位の決定に副次的に使われる。純粋なメタ情報は 項番1・2・9・22_2・79 のみ。

## 🗺️ 処理フロー概観

```
【入力：レセプト情報 + 患者情報】
    年齢 / 保険種別 / 入外区分 / 所得区分 / 患者状態 / 就学状況 / 多子世帯 / 受診日
        ↓
【判定①：制度有効期間チェック】（項番10・11）
        ↓
【判定②：適用可否チェック】（項番12〜30）
    → 不可 → スキップ
        ↓ 可
【判定③：助成タイミング条件チェック】（項番33〜39）
    初診時/入院初回/連続入院月数/受診回数
        ↓
【計算①：助成額 or 自己負担額の算出】（項番32・41〜50）
    入院/外来 × 全額/固定額/割合/分数
        ↓
【計算②：食事・生活療養費の算出】（項番51・80〜82）
        ↓
【計算③：上限額の適用】（項番53〜86）
    1日/1回/1月 × 入院/外来/食事療養
    端数処理・多数回該当・入院日数条件
        ↓
【計算④：累計処理】（項番40・87〜91）
    保険変更前・公費番号変更前の負担額を累計
        ↓
【出力：公費負担額 / 患者自己負担額 / 助成方法】
```
