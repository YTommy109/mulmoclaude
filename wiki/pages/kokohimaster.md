# 国公費マスタ（地単公費）仕様

都道府県・市区町村が実施する地域単独公費（地単公費）の助成ルールを定義するマスタ。
社会保険診療報酬支払基金の共通算定モジュールが参照し、患者負担額・公費負担額を算出する。

出典：[社会保険診療報酬支払基金 地単公費マスタ入力項目一覧（2026-03-30版）](https://www.ssk.or.jp/seikyushiharai/titansys/index.files/siryo2_20260330.pdf)

---

## 1. 構造図

### 1.1 データ構造の全体像

> **マスタの役割について**
> このマスタには性質の異なる2種類の項目が混在している。
>
> - **受給資格判定の根拠定義**（適用可否ブロック）：年齢・所得・保険種別・患者状態など。
>   これらは自治体が「誰に受給者証を交付するか」を決める条件定義であり、
>   請求時には**受給者証の保有**という事実に集約されている。
>   請求計算システムがこれらを直接参照するのは、保険種別・診療区分のような
>   レセプト情報と突合しやすい項目に限られる。
>
> - **負担計算の規則定義**（計算ブロック）：助成方法・タイミング条件・負担パターン・上限額など。
>   これらは受給資格が確認された後に、実際の金額を算出するために使用される。

```
国公費マスタ（地単公費）
│
├─ 【メタ情報】─────────────── 事業の識別・管理情報。計算には使用しない。
│   ├─ 事業名（正式名称 / 略称）
│   ├─ 管轄（都道府県番号 / 市区町村コード）  ※①
│   ├─ 分類番号（最大3つ、優先度順）          ※①
│   ├─ 公費負担者番号
│   ├─ 事業内区分コード
│   └─ 参考情報
│
├─ 【有効期間】─────────────── 制度（マスタ行）が有効な期間。
│   ├─ 有効開始年月日
│   └─ 有効終了年月日
│
├─ 【受給資格判定の根拠定義】── 自治体が受給者証を交付する条件の定義。     ※②
│   ├─ 年齢要件
│   │   ├─ 下限（基準日の種類 ＋ 年齢）
│   │   └─ 上限（基準日の種類 ＋ 年齢）
│   ├─ 診療区分（入院 / 外来）               ← 請求時にも参照
│   ├─ 保険種別（社保 / 国保 / 後期高齢）    ← 請求時にも参照
│   ├─ ひとり親等識別（児童 / 監護者）
│   ├─ 所得区分
│   ├─ 患者の状態（高額かつ長期 / 重症認定 / 重度継続 / 人工呼吸器）
│   ├─ 就学状況
│   ├─ 多子世帯（子ども人数 / 出生順位）
│   └─ 条件結合ロジック（AND / OR / 混在）
│
└─ 【負担計算の規則定義】────── 受給資格確認後、金額算出に使用される。
    ├─ 助成方法（現物給付の種類）
    ├─ 条件方向（助成条件 / 自己負担条件）
    ├─ タイミング条件
    │   ├─ 初診時のみ
    │   ├─ 入院初回時のみ
    │   ├─ 連続入院（継続性 / 開始月 / 終了月）
    │   └─ 複数回受診（回数の下限 / 上限）
    ├─ 負担パターン
    │   ├─ 入院（パターン種別 ＋ 金額 / 割合% / 分数）
    │   └─ 外来（パターン種別 ＋ 金額 / 割合% / 分数）
    ├─ 食事・生活療養費の助成範囲
    │   ├─ 入院時食事療養費
    │   └─ 入院時生活療養費（一括 or 個別）
    │       ├─ 生活食事療養費
    │       └─ 生活環境療養費
    ├─ 上限額
    │   ├─ 識別（公費負担上限 / 自己負担上限）
    │   ├─ 入院外来合算フラグ
    │   ├─ 多数回該当（入院 / 外来）
    │   ├─ 端数処理（単位 / 方法）
    │   ├─ 入院分（1日当たり・日数条件 / 1月当たり / 多数回該当）
    │   ├─ 外来分（1回当たり・回数上限 / 1日当たり・日数上限 / 1月当たり / 多数回該当）
    │   └─ 食事・生活療養費分（種別ごとに 1日当たり / 1月当たり）
    ├─ 負担額累計処理
    │   ├─ 保険制度変更時の扱い
    │   ├─ 公費負担者番号変更時の扱い
    │   └─ 累計法別番号（最大4件）
    ├─ 他公費連携
    │   ├─ 他公費の自己負担額を助成するか
    │   └─ 対象事業の指定（最大3件）
    └─ 公費適用優先順位
```

> ※① 市区町村コード未設定・分類番号の内容は、**計算の優先順位判定に副次的に使用される**。
> 純粋なメタ情報は 事業名 / 公費負担者番号 / 事業内区分コード / 参考情報 のみ。
>
> ※② 診療区分・保険種別は、受給者証発行条件としての定義に加え、
> 請求時のレセプト情報との突合にも使用される。

### 1.2 処理フロー

```
【受給者証発行フロー（自治体側）】

  患者の申請
        │
        ▼
  受給資格判定（マスタの「受給資格判定の根拠定義」を参照）
  年齢 / 所得 / 保険種別 / 患者状態 / 就学 / 多子 など
        │ 要件を満たす
        ▼
  受給者証の交付（公費負担者番号・有効期間・所得区分等を記載）


【請求・患者負担計算フロー（医療機関 / 算定モジュール側）】

  前提：患者が有効な受給者証を保有している
        │
        ▼
  入力（受給者証情報 ＋ レセプト情報）
  ・公費負担者番号（受給者証記載）
  ・診療年月日 ・診療区分（入院 / 外来）
  ・保険種別   ・受診回数 ・入院継続月数
        │
        ▼
  ① マスタ照合 ─── 該当なし ────────────────────→ 適用なし
  　 公費負担者番号でマスタを特定。
  　 同一番号で複数行ある場合は事業内区分コードで絞り込み。
        │ 照合成功
        ▼
  ② 制度有効期間チェック ─── 期間外 ─────────────→ 適用なし
  　 レセプトの診療年月日がマスタの有効期間内か。
        │ 期間内
        ▼
  ③ 診療区分・保険種別チェック ── 不一致 ──────────→ 適用なし
  　 レセプトの入院/外来区分・保険種別が
  　 マスタの対象範囲に含まれるか。
        │ 一致
        ▼
  ④ タイミング条件チェック ── 不一致 ────────────→ 当該請求は対象外
  　 初診時のみ / 入院初回時のみ /
  　 連続入院月数 / 同月受診回数
        │ 一致
        ▼
  ⑤ 負担パターン適用
  　 入院 / 外来 × 全額 / 固定額 / 割合 / 分数
        │
        ▼
  ⑥ 食事・生活療養費の助成適用
  　 食事療養費 / 生活食事療養費 / 生活環境療養費
        │
        ▼
  ⑦ 上限額の適用
  　 1日 / 1回 / 1月 × 端数処理 × 多数回該当
        │
        ▼
  ⑧ 累計処理
  　 保険変更前 / 公費番号変更前の患者負担額を通算
        │
        ▼
出力：公費負担額 / 患者自己負担額 / 助成方法
```
